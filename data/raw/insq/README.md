# INSQ Transcripts And Enriched Metadata

This folder contains the INSQ transcript files and enriched metadata used by the CIG study. INSQ text is approved for redistribution in this release.

Each conversation has a transcript CSV and a corresponding metadata JSON file named by conversation id, for example:

- `insq_2228.csv`
- `insq_2228_meta.json`

## Transcript CSV Files

The CSV files contain the utterance-level transcript data used by the pipeline, including conversation/topic identifiers, utterance indices/ids, speaker/role fields, text, and stage markers.

## Metadata JSON Files

The `*_meta.json` files are not merely raw corpus metadata. They are enriched pipeline artifacts produced during the study. Depending on the conversation, they include:

- `topic`, `moderator`, `speakers`, and `speaker_options`: conversation-level metadata.
- `intro_summary`: generated summary of the introduction/opening context.
- `segmentation`: segment boundaries and confidence information.
- `segmentation.segments[*].prior_summary`: prior-context summary available before a segment.
- `segmentation.segments[*].summary_ratings`: GPT-5 utterance ratings under different summary/context variants, typically `prior_summary`, `direct_summary`, `aspect_summary`, and `no_summary`.
- `segmentation.segments[*].summary_scores`: comparison scores for the summary/context variants.
- `segmentation.segments[*].full_context_rating`: GPT-5 ratings when using the full prior transcript context.
- `prior_memory`: memory-store items generated by the semantic-memory pipeline.
- `backend_config`: model/backend settings used for parts of the pipeline when recorded.

Dense embedding vectors were removed from the public `prior_memory` entries to keep the repository lightweight. The memory text, metadata, generated summaries, and model ratings remain available. For local reruns, `scripts/archive_memory_embeddings.py` extracts the original vectors from ignored local metadata into `data/archive_local/embeddings/insq_prior_memory_embeddings.jsonl`; scripts that rehydrate memory stores look there first and regenerate missing vectors if the sidecar is absent.

The generated ratings are model annotations used for analysis and ablations, not human gold labels. Human individual and consensus labels are released separately under `data/release/`.

## Source And Citation

The INSQ files are derived from the ConvoKit Intelligence Squared Debates Corpus:

- https://convokit.cornell.edu/documentation/iq2.html
- ConvoKit download name: `iq2-corpus`

The ConvoKit page describes the original dataset as compiled by and featured in Zhang, Justine, et al. "Conversational Flow in Oxford-Style Debates." NAACL, 2016.

The ConvoKit documentation page does not state a separate corpus-specific license on that page. Please cite the ConvoKit corpus page, the original Zhang et al. paper, and this repository, and follow any terms associated with the source corpus.
